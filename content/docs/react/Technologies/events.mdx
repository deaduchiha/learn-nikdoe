# 🖱 React Events — سیستم رویداد مصنوعی

در HTML/JS ساده، ممکن است چندین native event listener اضافه کنید:

```js
button.addEventListener("click", handleClick);
```

اما در **React**، شما می‌نویسید:

```jsx
<button onClick={handleClick}>Click Me</button>
```

پس تفاوت چیست؟

React از یک **سیستم رویداد مصنوعی** استفاده می‌کند — لایه هوشمندی که دور رویدادهای native مرورگر می‌پیچد.

---

## 🧠 آنچه یاد خواهید گرفت

- رویدادهای مصنوعی چیست
- چرا React از آنها استفاده می‌کند
- چگونه React عملکرد را با یک listener بهبود می‌دهد
- چگونه از رویدادها در کامپوننت‌ها استفاده کنید
- تفاوت‌ها با رویدادهای native

---

## 🔹 رویداد مصنوعی چیست؟

یک **رویداد مصنوعی** یک **wrapper دور یک رویداد native** است که به همان روش **در تمام مرورگرها** کار می‌کند.

React صدها listener اضافه نمی‌کند — در عوض، از **یک global listener** در root (مانند `document`) استفاده می‌کند و رویدادها را به صورت داخلی delegate می‌کند.

✅ مزایا:

- رفتار یکپارچه و cross-browser
- کارآمد از نظر حافظه — listener های کمتر
- event pooling داخلی (تا React 17)
- بهینه‌سازی عملکرد برای اپ‌های بزرگ

---

## 🧪 مثال: استفاده از رویدادهای مصنوعی

```jsx
function App() {
  const handleClick = (event) => {
    console.log("Clicked!", event.type); // رویداد مصنوعی
  };

  return <button onClick={handleClick}>Click Me</button>;
}
```

🧠 `event` یک **SyntheticEvent** است، نه یک رویداد خام مرورگر.

---

## 🔸 چگونه React به رویدادها گوش می‌دهد

به جای انجام این کار:

```html
<button onclick="..."></button>
<!-- یک listener برای هر دکمه -->
```

React این کار را پشت صحنه انجام می‌دهد:

```
document.addEventListener("click", ...) // فقط یک بار!
```

سپس تشخیص می‌دهد که کدام کامپوننت واقعاً رویداد را trigger کرده است. این **event delegation** نامیده می‌شود.

---

## 🔸 رویدادهای پشتیبانی شده

React از بیشتر رویدادهای رایج پشتیبانی می‌کند:

- **Mouse**: `onClick`، `onDoubleClick`، `onMouseEnter`
- **Keyboard**: `onKeyDown`، `onKeyPress`، `onKeyUp`
- **Form**: `onChange`، `onSubmit`، `onFocus`، `onBlur`
- **Touch**: `onTouchStart`، `onTouchEnd`، `onTouchMove`

✅ این‌ها در تمام مرورگرها یکسان کار می‌کنند — به لطف لایه رویداد مصنوعی.

---

## 🔹 ویژگی‌های شیء رویداد

```jsx
function Input() {
  const handleChange = (e) => {
    console.log(e.target.value); // دقیقاً مثل JS معمولی
  };

  return <input onChange={handleChange} />;
}
```

رویدادهای مصنوعی از رویدادهای native تقلید می‌کنند، بنابراین شما هنوز دسترسی دارید به:

- `e.target`
- `e.preventDefault()`
- `e.stopPropagation()`
- `e.type`، `e.currentTarget` و غیره

---

## ⚠️ نکته در مورد React 17+

در React 17+، رویدادها **به root کانتینر bubble می‌کنند**، نه به `document`.

این به شما می‌دهد:

- کنترل بهتر با چندین اپ React در یک صفحه
- رفتار بیشتر شبیه native
- هنوز یک listener، اما محدودتر

---

## 🧪 تکالیف تمرینی

۱. ✅ یک کامپوننت `Button` بسازید و شیء رویداد را داخل `onClick` log کنید.
۲. ✅ سعی کنید از `e.preventDefault()` روی یک لینک استفاده کنید تا از navigate کردن آن جلوگیری کنید.
۳. ✅ یک `Input` بسازید که `e.target.value` را در هر keystroke log کند.

## 🧠 خلاصه

| مفهوم                      | توضیحات                                                        |
| -------------------------- | -------------------------------------------------------------- |
| رویداد مصنوعی              | wrapper cross-browser React برای رویدادهای native              |
| یک Global Listener         | React از delegation استفاده می‌کند، نه handler های per-element |
| API یکپارچه                | در Chrome، Firefox، Safari یکسان کار می‌کند                    |
| پشتیبانی از بیشتر رویدادها | Keyboard، Mouse، Touch، Forms، Clipboard و غیره                |
