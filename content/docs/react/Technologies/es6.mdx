## 🧠 اهداف مهارتی

- درک `let`، `const` و محدوده بلوکی
- استفاده از توابع arrow برای callback های تمیز
- ساخت ماژول‌های قابل استفاده مجدد با `import/export`
- ساختاردهی کد با کلاس‌ها و متدها
- نوشتن رشته‌های قابل خواندن با استفاده از template literals

---

## 🧩 مفاهیم و مثال‌ها

### 🔹 `let` و `const`

````js
let age = 20;          // مقدار قابل تغییر است
const siteName = "Nikode"; // مقدار قابل تغییر نیست


> ✅ به طور پیش‌فرض از `const` استفاده کنید. فقط در صورتی که قصد تغییر مجدد مقدار را دارید، به `let` تغییر دهید.

---

### 🔹 توابع Arrow

```js
const greet = (name) => {
  return `Hello, ${name}!`;
};
````

✅ کوتاه‌تر است و از مشکلات `this` در callback ها جلوگیری می‌کند.

---

### 🔹 Template Literals

```js
const lang = "JavaScript";
console.log(`I love ${lang}!`);
```

دیگر نیازی به `+` برای اتصال رشته‌ها نیست. فقط از backtick `` ` `` و `${}` استفاده کنید.

---

### 🔹 ماژول‌ها: `import` و `export`

**math.js**

```js
export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;
```

**main.js**

```js
import { add, subtract } from "./math.js";

console.log(add(10, 5)); // 15
console.log(subtract(10, 5)); // 5
```

> ماژول‌ها = کد تمیزتر و قابل استفاده مجدد در فایل‌های مختلف.

---

### 🔹 کلاس‌ها

```js
class Person {
  constructor(name) {
    this.name = name;
  }

  greet() {
    return `Hi, I'm ${this.name}`;
  }
}

const user = new Person("Nikode");
console.log(user.greet());
```

> کلاس‌ها را مانند نقشه‌های ساختمانی برای اشیاء با منطق در نظر بگیرید.
