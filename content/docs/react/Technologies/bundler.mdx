## 🔹 Bundler چیست؟

یک **bundler** فایل‌های متعدد اپلیکیشن شما (JS، CSS، تصاویر و غیره) را می‌گیرد و **آنها را در چند فایل خروجی بهینه‌سازی شده ترکیب می‌کند**.

```txt
input/
├─ App.jsx
├─ styles.css
├─ logo.png
└─ utils/
   └─ helpers.js

▼

output/
├─ bundle.js
├─ style.css
```

✅ مزایا:

- درخواست‌های HTTP کمتر
- فایل‌های فشرده و بهینه‌سازی شده
- زمان بارگذاری سریع‌تر
- پشتیبانی از ابزارهای مدرن مانند JSX و ماژول‌ها

---

## 🔧 Bundler های محبوب در یک نگاه

| ابزار       | موتور                       | ویژگی‌های کلیدی                                 |
| ----------- | --------------------------- | ----------------------------------------------- |
| **Vite**    | از `esbuild` استفاده می‌کند | سرور توسعه فوق‌العاده سریع، hot reload          |
| **esbuild** | با Go نوشته شده             | bundler JS/TS فوق‌العاده سریع، zero config      |
| **Webpack** | مبتنی بر پلاگین             | قدرتمند و انعطاف‌پذیر، استفاده شده توسط Next.js |

---

## ⚡ Vite (پیش‌فرض برای React مدرن)

Vite یک bundler نسل جدید است که از **esbuild** در پس‌زمینه برای توسعه استفاده می‌کند.

```bash
npm create vite@latest
```

### 🔥 ویژگی‌ها:

- راه‌اندازی فوری (بدون bundling در توسعه)
- HMR (Hot Module Reloading)
- پشتیبانی بومی ESM
- با React، Vue، Svelte و غیره عالی کار می‌کند

---

## 🛠 esbuild (پادشاه سرعت)

esbuild یک bundler و minifier JS فوق‌العاده سریع است.

```bash
npx esbuild app.jsx --bundle --outfile=bundle.js
```

✅ اغلب **پشت صحنه** در ابزارهایی مانند این استفاده می‌شود:

- Vite (برای توسعه)
- Bun
- Snowpack

---

## 🏗 Webpack (غول قدیمی)

Webpack bundler سنتی است که در **Create React App** و **Next.js** استفاده می‌شود.

```bash
npx create-react-app my-app
```

### ویژگی‌های کلیدی:

- بسیار قابل تنظیم با loader ها و پلاگین‌ها
- تصاویر، CSS، SVG و موارد دیگر را مدیریت می‌کند
- بهینه‌سازی‌های قدرتمند تولید

> 🧠 انعطاف‌پذیرتر، اما همچنین پیچیده‌تر برای تنظیم دستی.

## 📦 چگونه Bundler ها کار می‌کنند (ساده‌شده)

```
┌────────────┐
│  کد شما    │  (JSX، CSS، TS، تصاویر...)
└─────┬──────┘
      ▼
┌──────────────┐
│   ترنسپایلر │  (Babel، esbuild)
└─────┬────────┘
      ▼
┌─────────────┐
│   Bundler   │  (Vite، Webpack)
└─────┬───────┘
      ▼
┌────────────────────┐
│  Bundle خروجی      │ → تحویل به مرورگر
└────────────────────┘
```

## 🧪 تکالیف تمرینی

۱. ✅ یک اپ React جدید با استفاده از **Vite** بسازید و `vite.config.js` را بررسی کنید.
۲. ✅ از `esbuild` از CLI برای bundle کردن یک فایل JS ساده استفاده کنید.
۳. ✅ اندازه bundle های خروجی بین CRA و Vite را مقایسه کنید.
