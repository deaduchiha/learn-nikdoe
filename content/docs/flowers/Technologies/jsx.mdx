## 🔹 What Is JSX?

JSX looks like HTML, but it's actually **JavaScript**. This code:

```jsx
const element = <h1>Hello, Nikode!</h1>;
```

…gets compiled into this:

```js
const element = React.createElement("h1", null, "Hello, Nikode!");
```

💡 **Why JSX?**

- It's more **readable** and looks like the UI you build
- It's **faster** to write and understand than `React.createElement()`
- It supports **JavaScript logic** inside `{}` braces

---

## 🔹 JSX in Action

```jsx
function Welcome(props) {
  return <h1>Hello, {props.name}!</h1>;
}

<Welcome name="Zahra" />;
```

🧠 Inside the `<h1>`, `{props.name}` is a **JavaScript expression** — not a string!

## 🔸 Embedding Expressions in JSX

You can use **any JS expression** inside curly braces `{}`:

```jsx
const age = 20;
const message = <p>You are {age >= 18 ? "an adult" : "a minor"}.</p>;
```

✅ Valid expressions: variables, functions, conditionals, math, etc.

## ⚠️ JSX Rules You Must Know

### 1. **One Parent Element**

This is invalid:

```jsx
return <h1>Title</h1><p>Text</p>; ❌
```

This is valid:

```jsx
return (
  <>
    <h1>Title</h1>
    <p>Text</p>
  </>
); ✅
```

> Use a wrapping `<div>` or a **fragment** (`<>...</>`) if needed.

---

### 2. **Use `className` Instead of `class`**

```jsx
<p className="intro">Welcome!</p> ✅
```

> `class` is a reserved word in JS, so JSX uses `className`.

---

### 3. **Self-Closing Tags**

Always close single tags:

```jsx
<img src="logo.png" /> ✅
<input /> ✅
```

---

## 🔸 Conditional Rendering

Show elements only when needed:

```jsx
{
  isLoggedIn ? <p>Welcome back!</p> : <p>Please log in.</p>;
}
```

Or:

```jsx
{
  showNotice && <div className="notice">New update available!</div>;
}
```

---

## 🔸 Lists in JSX

```jsx
const items = ["HTML", "CSS", "JS"];
return (
  <ul>
    {items.map((item) => (
      <li key={item}>{item}</li>
    ))}
  </ul>
);
```

🧠 **Always add `key`** to items in a list to help React track changes.
