# 🌳 1 · TRUNK — HTML & Accessibility

> **Scope:** This chapter focuses **only** on semantic HTML, logical document structure, and ARIA labelling—**no JavaScript, no interactive widgets**. Perfect for absolute beginners who need rock‑solid markup skills before moving on.

---

## 🏗️ 1. HTML Foundations

### 1.1 Minimal Document

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <title>My First Page</title>
  </head>
  <body>
    <h1>Hello, world!</h1>
  </body>
</html>
```

### 1.2 Tags, Elements & Attributes

- **Tag:** `<p>`
- **Element:** `<p>Hello</p>`
- **Attribute:** `<img src="cat.jpg" alt="Cute kitten" />`

### 1.3 Block vs. Inline Cheat‑Sheet

| Block Elements (start new line)             | Inline Elements (flow in text)    |
| ------------------------------------------- | --------------------------------- |
| `<div>` `<section>` `<header>` `<p>` `<ul>` | `<span>` `<a>` `<strong>` `<img>` |

### 1.4 Self‑Closing Tags

`<img>`, `<input>`, `<br>`, `<hr>`

\### 1.5 Nesting Rules & Validation

- Children sit _inside_ parents.
- Tags close in LIFO order.

> **Mini‑Drill:** Build a two‑section page with headings, paragraphs, list, and image—no CSS.

## 👓 2. Why Accessibility First?

- 20 % of users rely on assistive tech.
- Clean markup = better SEO.
- Retro‑fitting a11y is expensive; start right.

## 📖 3. Key Concepts & Examples

### 3.1 Semantic Landmarks

```html
<header>
  <h1>My Portfolio</h1>
  <nav aria-label="Primary">
    <ul>
      <li><a href="#projects">Projects</a></li>
      <li><a href="#contact">Contact</a></li>
    </ul>
  </nav>
</header>
<main id="content">
  <article>
    <h2 id="projects">Featured Project</h2>
    <p>Short description…</p>
  </article>
</main>
<footer>
  <p>© 2025 Your Name</p>
</footer>
```

**Guidelines**

- Only _one_ `<main>` per page.
- Use `<section>` or `<article>` when the block gets its own heading.

---

### 3.2 Headings Hierarchy

```html
<h1>Recipe Blog</h1>
<h2>Chocolate Cake</h2>
<h3>Ingredients</h3>
<h3>Steps</h3>
<h2>Lemon Tart</h2>
```

- One `<h1>` per page.
- Don’t skip levels (`h1`→`h3` = ❌).

---

### 3.3 Forms (Fully HTML)

```html
<form action="/subscribe" method="post" novalidate>
  <fieldset>
    <legend>Email Newsletter</legend>

    <label for="email">Email address</label>
    <input
      type="email"
      id="email"
      name="email"
      required
      aria-describedby="email-hint"
    />
    <p id="email-hint">We’ll never share your email.</p>

    <button>Subscribe</button>
  </fieldset>
</form>
```

**Best Practices**

- Connect every `<label>` to its control via `for`/`id`.
- Use `fieldset` + `legend` to group related inputs.
- Supplementary help? Use `aria-describedby`.

---

### 3.4 Alt Text

| Image Type      | Example                                                                         | Why                       |
| --------------- | ------------------------------------------------------------------------------- | ------------------------- |
| Informative     | `<img src="team.jpg" alt="Four team members smiling in front of whiteboard" />` | Conveys content           |
| Decorative      | `<img src="ornament.svg" alt="" role="presentation" />`                         | Ignored by screen‑readers |
| Functional/Icon | `<img src="download.svg" alt="Download PDF" />`                                 | Acts as button label      |

---

### 3.5 ARIA Labelling Patterns (No JS)

**Common cases and their cleanest solutions**

| Need                          | Attribute          | Minimal snippet                                                                                                                                                            |     |                    |                                         |
| ----------------------------- | ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --- | ------------------ | --------------------------------------- |
| Hidden label (icon button)    | `aria-label`       | `<button aria-label="Close">✖</button>`                                                                                                                                    |     |                    |                                         |
| Visible label outside control | `aria-labelledby`  | Use a nearby visible heading, e.g. `&lt;h2 id="pricing"&gt;Pricing&lt;/h2&gt;`, then reference it on the control: `&lt;table aria-labelledby="pricing"&gt;…&lt;/table&gt;` |     |                    |                                         |
| Extra description / hint      | `aria-describedby` | `<input aria-describedby="email-hint">`                                                                                                                                    |     | `aria-describedby` | `<input aria-describedby="email-hint">` |

> **Reading order for screen‑readers:** `aria-labelledby` → visible `<label>` → `aria-describedby`.

#### Full example for `aria-labelledby`

```html
<h2 id="pricing">Pricing</h2>
<table aria-labelledby="pricing">
  …
</table>
```

---

### 3.6 Static State Attributes Static State Attributes

Even without JS, static ARIA states communicate important info.

```html
<button aria-disabled="true">Coming Soon</button>
<progress value="70" max="100" aria-valuetext="70 percent"></progress>
```

---

### 3.7 Focus Styles (CSS‑only)

```css
:where(a, button, input):focus-visible {
  outline: 3px solid #004bcc;
  outline-offset: 2px;
}
```

Ensure every interactive element is keyboard‑accessible & visibly focused.
