## درک Git و GitHub

**Git** یک سیستم کنترل نسخه توزیع شده است که به ردیابی تغییرات در کد شما کمک می‌کند. **GitHub** پلتفرمی است که repository های Git را می‌زبانی می‌کند و همکاری بی‌نقص را امکان‌پذیر می‌سازد.

---

## نصب Git

Git را از [git-scm.com](https://git-scm.com/downloads) دانلود کنید.

- تنظیمات نصب پیش‌فرض را دنبال کنید.
- نصب خود را با تایپ کردن تأیید کنید:

```bash
git --version
```

---

## پیکربندی پایه

هویت خود را در Git پیکربندی کنید:

```bash
git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"
```

تنظیمات خود را بررسی کنید:

```bash
git config --list
```

---

## ایجاد اولین Repository

Repository خود را راه‌اندازی کنید:

```bash
mkdir my-first-project
cd my-first-project
git init
```

یک README اضافه کنید:

```bash
echo "# My First Project 🚀" > README.md
```

تغییرات را اضافه و commit کنید:

```bash
git add .
git commit -m "first commit"
```

---

## اتصال به GitHub

یک repository در GitHub ایجاد کنید:

- در [github.com](https://github.com) وارد شوید.
- روی "New Repository" کلیک کنید.
- repository خود را نام‌گذاری کنید و بقیه را در حالت پیش‌فرض بگذارید.

repository محلی خود را لینک کنید:

```bash
git remote add origin <github-repository-address>
```

کد خود را به GitHub push کنید:

```bash
git push -u origin main
```

---

## branch‌زنی و ادغام

یک branch جدید ایجاد کنید:

```bash
git checkout -b <branch-name>
```

تغییر branch ها:

```bash
git checkout main
```

ادغام branch ها:

```bash
git merge <branch-name>
```

حذف یک branch:

```bash
git branch -d <branch-name>
```

---

## مدیریت تعارض‌ها

تعارض‌ها زمانی رخ می‌دهند که branch ها با تغییرات همپوشان ادغام می‌شوند.

- Git تعارض‌ها را در فایل‌ها به وضوح علامت‌گذاری می‌کند.

نمونه نشانگرهای تعارض:

```
این تغییر branch فعلی شما است
```

با ویرایش دستی فایل و ذخیره آن حل کنید. سپس:

```bash
git add .
git commit -m "Resolve merge conflict"
```

## دستورات پیشرفته Git

### مشاهده تاریخچه:

```bash
# مشاهده commit های اخیر
git log --oneline

# مشاهده تغییرات در commit خاص
git show <commit-hash>

# مشاهده تفاوت بین branch ها
git diff main..<branch-name>
```

### بازگردانی تغییرات:

```bash
# بازگردانی فایل به آخرین commit
git checkout -- نام-فایل

# بازگردانی به commit خاص
git reset --hard <commit-hash>

# بازگردانی نرم (حفظ تغییرات)
git reset --soft <commit-hash>
```

### Stash کردن تغییرات:

```bash
# ذخیره تغییرات موقت
git stash

# مشاهده stash های موجود
git stash list

# اعمال آخرین stash
git stash pop

# حذف stash
git stash drop
```

### Tag کردن:

```bash
# ایجاد tag برای نسخه
git tag v1.0.0

# ایجاد tag با پیام
git tag -a v1.0.0 -m "نسخه ۱.۰.۰"

# مشاهده همه tag ها
git tag

# Push کردن tag ها
git push origin --tags
```

## بهترین شیوه‌ها

### پیام‌های commit خوب:

```bash
# خوب
git commit -m "feat: add authentication system"

# بهتر
git commit -m "feat: add authentication system with JWT

- Add authentication middleware
- Implement login/logout API
- Add validation for forms"
```

### انواع commit:

- `feat:` - ویژگی جدید
- `fix:` - رفع باگ
- `docs:` - تغییرات مستندات
- `style:` - تغییرات فرمت
- `refactor:` - بازسازی کد
- `test:` - اضافه کردن تست
- `chore:` - کارهای نگهداری

### کار با branch ها:

```bash
# ایجاد branch از branch خاص
git checkout -b feature/new-feature main

# تغییر نام branch
git branch -m old-name new-name

# مشاهده branch های remote
git branch -r

# حذف branch remote
git push origin --delete <branch-name>
```

## . کار با Remote

### اضافه کردن remote جدید:

```bash
git remote add upstream https://github.com/original/repo.git
```

### مشاهده remote ها:

```bash
git remote -v
```

### دریافت تغییرات:

```bash
# دریافت تغییرات بدون merge
git fetch origin

# دریافت و merge تغییرات
git pull origin main

# دریافت تغییرات از upstream
git fetch upstream
```

### Push کردن تغییرات:

```bash
# Push branch جدید
git push -u origin branch-جدید

# Push همه branch ها
git push --all origin

# Force push (مراقب باشید!)
git push --force origin main
```

---

## . حل مشکلات رایج

### فراموش کردن commit:

```bash
# اضافه کردن فایل فراموش شده
git add فایل-فراموش-شده
git commit --amend --no-edit
```

### تغییر آخرین commit:

```bash
# تغییر پیام آخرین commit
git commit --amend -m "پیام جدید"

# اضافه کردن فایل به آخرین commit
git add فایل-جدید
git commit --amend --no-edit
```

### بازگردانی فایل حذف شده:

```bash
# مشاهده فایل‌های حذف شده
git log --diff-filter=D --summary

# بازگردانی فایل
git checkout <commit-hash> -- مسیر/فایل
```

## . کار با GitHub

### Fork کردن repository:

1. به repository مورد نظر بروید
2. روی "Fork" کلیک کنید
3. repository را به حساب خود fork کنید

### Pull Request:

1. تغییرات خود را در branch جدید ایجاد کنید
2. تغییرات را push کنید
3. روی "New Pull Request" کلیک کنید
4. توضیحات تغییرات را بنویسید

### Issues:

- برای گزارش باگ‌ها استفاده کنید
- برای درخواست ویژگی‌های جدید استفاده کنید
- برای سوالات و بحث استفاده کنید

## . تکالیف: تمرینات عملی

این کارها را برای تقویت مهارت‌های خود انجام دهید:

### ۱. ایجاد و Clone:

- یک repository گیت‌هاب ایجاد کنید، آن را محلی clone کنید، فایل‌ها اضافه کنید، commit کنید و push کنید.

### ۲. branch‌زنی و ادغام:

- دو branch ایجاد کنید، تغییرات مختلف انجام دهید و ادغام را تمرین کنید.

### ۳. شبیه‌سازی تعارض:

- با تغییر همان خط در دو branch تعارض شبیه‌سازی کنید، سپس ادغام کنید و حل کنید.

### ۴. سناریوی دنیای واقعی:

- تصور کنید با دوست خود همکاری می‌کنید. شبیه‌سازی کنید دو نفر تغییرات متضاد push می‌کنند، این تعارض‌ها را حل کنید و فرآیند را به وضوح مستند کنید.

### ۵. تمرین دستورات پیشرفته:

- از stash برای ذخیره تغییرات موقت استفاده کنید
- tag برای نسخه‌گذاری ایجاد کنید
- تاریخچه commit ها را بررسی کنید

## . نکات پیشرفته

### Git Hooks:

```bash
# مشاهده hooks موجود
ls .git/hooks/

# ایجاد hook سفارشی
touch .git/hooks/pre-commit
chmod +x .git/hooks/pre-commit
```

### Git Aliases:

```bash
# ایجاد alias برای دستورات پرکاربرد
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status
```

### Git Workflow:

1. **Feature Branch Workflow**: برای پروژه‌های کوچک
2. **Gitflow**: برای پروژه‌های بزرگ
3. **Trunk-Based Development**: برای تیم‌های بزرگ
