> **هدف ساده**  
> از هیچ شروع کنید، گام‌های کوچک یاد بگیرید، با صفحه‌ای تمام کنید که برای _همه_ مردم کار می‌کند.

---

# 🌱 سطح ۱ — مبتدی

### ۱.۱ دسترسی‌پذیری چیست؟

- همه می‌توانند بخوانند، کلیک کنند و از سایت استفاده کنند.
- با صفحه‌خوان‌ها، کیبورد، صدا و حالت کنتراست بالا کار می‌کند.

### ۱.۲ چرا مهم است؟

| دلیل      | توضیح ساده                                         |
| --------- | -------------------------------------------------- |
| **مردم**  | از هر ۵ بازدیدکننده، ۱ نفر به ابزار کمک نیاز دارد. |
| **جستجو** | HTML تمیز به گوگل کمک می‌کند شما را پیدا کند.      |
| **پول**   | تعمیر زودهنگام = ارزان. تعمیر دیرهنگام = گران.     |

### ۱.۳ پیروزی‌های سریع

۱. **متن جایگزین** روی هر تصویر.

```html
<img src="سگ.jpg" alt="سگ قهوه‌ای که در پارک می‌دود" />
```

۲. **عناوین** به ترتیب.

```html
<h1>عنوان</h1>
<h2>بخش</h2>
```

۳. **برچسب** برای هر ورودی.

```html
<label for="نام">نام</label> <input id="نام" />
```

۴. **کنتراست**: متن تیره روی پس‌زمینه روشن (نسبت ≥ ۴.۵:۱).

### ۱.۴ مثال کامل

```html
<!DOCTYPE html>
<html lang="fa" dir="rtl">
  <head>
    <meta charset="utf-8" />
    <title>صفحه سلام</title>
  </head>
  <body>
    <a href="#اصلی" class="لینک-پرش">پرش به محتوا</a>

    <header>
      <h1>سلام، دنیا!</h1>
    </header>

    <main id="اصلی">
      <p>اولین صفحه دسترسی‌پذیر من.</p>

      <img src="سلفی.jpg" alt="شخص خندان با هدفون" />

      <form>
        <label for="ایمیل">ایمیل</label>
        <input id="ایمیل" type="email" required />
        <button>ثبت‌نام</button>
      </form>
    </main>
  </body>
</html>
```

### ۱.۵ تکلیف

- متن جایگزین به سه تصویر روی هر صفحه‌ای که ساختید اضافه کنید.
- ترتیب عناوین را با افزونه مرورگر "طرح کلی HTML" بررسی کنید.

# 🌿 سطح ۲ — متوسط

### ۲.۱ نشانه‌های صفحه

```html
<header>…</header>
<nav aria-label="منوی اصلی">…</nav>
<main id="اصلی">…</main>
<footer>…</footer>
```

این تگ‌ها به کاربران صفحه‌خوان اجازه می‌دهند سریع بپرند.

### ۲.۲ لینک پرش (کمک‌کننده کیبورد)

```html
<a href="#اصلی" class="لینک-پرش">پرش به محتوا</a>
```

آن را در بالاترین قسمت قرار دهید. فقط هنگام فوکوس با CSS نمایش دهید.

### ۲.۳ فرم‌های بهتر

```html
<form novalidate>
  <fieldset>
    <legend>با من تماس بگیرید</legend>

    <label for="پیام">پیام</label>
    <textarea id="پیام" required></textarea>

    <button>ارسال</button>
  </fieldset>
</form>
```

**ایده‌های کلیدی**

- برچسب _باید_ به کنترل اشاره کند (`for` ↔ `id`).
- ورودی‌های مرتبط را با `<fieldset>` + `<legend>` گروه‌بندی کنید.
- از ویژگی‌های HTML مثل `required` و `type="email"` استفاده کنید.

### ۲.۴ فوکوس قابل مشاهده

```css
:focus-visible {
  outline: 3px solid #0033ff;
}
```

### ۲.۵ تکلیف

- یک "نمونه کار" ساده با header، nav، main و footer بسازید.
- لینک پرش اضافه کنید.
- فقط با کیبورد تست کنید (Tab، Shift+Tab، Enter).

# 🌳 سطح ۳ — پیشرفته

### ۳.۱ پیام‌های زنده (بدون تازه‌سازی)

```html
<p aria-live="polite" id="وضعیت">پیام جدیدی نیست</p>
```

اگر جاوااسکریپت `#وضعیت` را به‌روزرسانی کند، صفحه‌خوان تغییر را اعلام می‌کند.

### ۳.۲ الگوی آکاردئون

```html
<button aria-expanded="false" aria-controls="پنل۱" class="آکاردئون">
  سیاست بازپرداخت چیست؟
</button>
<div id="پنل۱" hidden>
  <p>بازپرداخت کامل در ۳۰ روز.</p>
</div>
```

```js
const btn = document.querySelector(".آکاردئون");
btn.addEventListener("click", () => {
  const panel = document.getElementById(btn.getAttribute("aria-controls"));
  const open = btn.getAttribute("aria-expanded") === "true";
  btn.setAttribute("aria-expanded", !open);
  panel.hidden = open;
});
```

### ۳.۳ مراحل تست

۱. **کیبورد** — آیا می‌توانید به همه چیز برسید؟ حلقه فوکوس قابل مشاهده است؟
۲. **ابزارهای خودکار** — Lighthouse را اجرا کنید → تب دسترسی‌پذیری.
۳. **صفحه‌خوان** — از VoiceOver (مک) یا NVDA (ویندوز) استفاده کنید و عناوین را بخوانید.

### ۳.۴ تکلیف

- یک FAQ آکاردئون به نمونه کارتان اضافه کنید.
- یک منطقه زنده اضافه کنید که بعد از ارسال فرم تماس "ارسال شد!" بگوید.

---

# ✅ لیست بهترین شیوه‌ها

۱. **ابتدا از تگ‌های HTML استفاده کنید**، ARIA فقط در صورت نیاز.
۲. فقط **یک `<h1>`** در هر صفحه.
۳. **حلقه فوکوس را بدون اضافه کردن استایل جدید حذف نکنید**.
۴. کنتراست رنگ را قوی نگه دارید.
۵. زودهنگام تست کنید، زودهنگام تعمیر کنید.

# 🎓 پروژه نهایی

یک سایت **"درباره من"** بسازید که _همه_ مهارت‌ها را نشان دهد:

- **مبتدی**: متن جایگزین، عناوین، برچسب‌های فرم.
- **متوسط**: نشانه‌ها، لینک پرش، فوکوس قابل مشاهده.
- **پیشرفته**: FAQ آکاردئون، پیام وضعیت زنده.

آن را منتشر کنید (GitHub Pages خوب کار می‌کند) و از یک دوست بخواهید:

۱. فقط با کیبورد ناوبری کند.
۲. صفحه‌خوان را روشن کند و عناوین شما را بخواند.
