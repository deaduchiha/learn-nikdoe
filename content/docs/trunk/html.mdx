# 🌳 تنه — مسیر اصلی

در زیر **نقشه راه دقیق** است که ما در **نیکود** دنبال می‌کنیم. به ترتیب بچسبید؛ هر "برگ" بر روی قبلی ساخته می‌شود. وقتی تمام شدید، یک مینی‌سایت کاملاً دسترسی‌پذیر منتشر خواهید کرد—**بدون یک خط جاوااسکریپت.**

> **نکته حرفه‌ای:** تمام مخازن دمو و ویدیوهای راه‌حل در انتهای هر مرحله لینک شده‌اند—انحصاراً در **نیکود**.

## 🏗️ مرحله ۱ · پایه‌های HTML

### ۱.۱ سند حداقلی

```html
<!DOCTYPE html>
<html lang="fa" dir="rtl">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>اولین صفحه من</title>
  </head>
  <body>
    <h1>سلام، دنیا!</h1>
    <p>به HTML معنادار خوش آمدید.</p>
  </body>
</html>
```

**چک‌لیست**

- `<!DOCTYPE html>` HTML مدرن را اعلام می‌کند.
- `<html lang="fa" dir="rtl">`: به مرورگرها و صفحه‌خوان‌ها می‌گوید صفحه فارسی است.
- `<meta charset="utf-8" />`: اطمینان می‌دهد کاراکترها درست نمایش داده شوند.
- `<meta name="viewport" content="width=device-width, initial-scale=1" />`: متن موبایل را خوانا نگه می‌دارد.
- یک—و فقط یک—`<h1>` در هر صفحه. ✨
- در **لینت نیکود** اعتبارسنجی می‌شود (تعمیر خودکار داخلی).

**بهترین شیوه**: همیشه با این قالب شروع کنید. حتی پروژه‌های پایه از تنظیمات charset، viewport و زبان بهره‌مند می‌شوند.

### ۱.۲ تگ‌ها، عناصر و ویژگی‌ها

| مفهوم       | مثال                                          | نکات کلیدی                                                        |
| ----------- | --------------------------------------------- | ----------------------------------------------------------------- |
| **تگ**      | `<p>`                                         | نام داخل براکت‌های زاویه‌دار، مثل `p`، `img`، `a`، `div`.         |
| **عنصر**    | `<p>سلام</p>`                                 | تگ باز + محتوا + تگ بسته. برخی تگ‌ها خودبسته هستند (مثل `<img>`). |
| **ویژگی**   | `<img src="گربه.jpg" alt="بچه گربه بامزه" />` | اطلاعات اضافی داخل تگ باز: `src`، `alt`، `href`، `id`، `class`.   |
| **تگ خالی** | `<br>` `<img>` `<input>` `<hr>`               | عناصری که محتوا را نمی‌پیچند—فقط `<br />` یا `<hr />` برای یک خط. |

> **نکته برای خوانندگان غیر انگلیسی زبان:** تگ‌ها را به عنوان "برچسب‌های نام"، عناصر را به عنوان "قطعات" و ویژگی‌ها را به عنوان "جزئیات" که به یک قطعه متصل می‌کنید در نظر بگیرید.

---

### ۱.۳ چیت‌شیت بلوک در مقابل درون‌خطی

| **بلوک (خط جدید شروع می‌کند)**                          | **درون‌خطی (با متن جریان دارد)**         |
| ------------------------------------------------------- | ---------------------------------------- |
| `<header>` `<section>` `<article>` `<p>` `<ul>` `<div>` | `<span>` `<a>` `<strong>` `<em>` `<img>` |

- عناصر **بلوک** عرض کامل می‌گیرند و روی خط جدید شروع می‌شوند.
- عناصر **درون‌خطی** داخل یک خط متن قرار می‌گیرند.

---

### ۱.۴ تگ‌های خودبسته (خالی)

- `<img src="عکس.jpg" alt="توضیح" />`
- `<input type="text" name="نام" />`
- `<br />` (شکستن خط)
- `<hr />` (خط افقی)

> **به یاد داشته باشید**: تگ‌های خالی متن را نمی‌پیچند. آنها همیشه بلافاصله تمام می‌شوند (`/>`).

---

### ۱.۵ قوانین تودرتو و اعتبارسنجی

- **تودرتو درست**: فرزندان _داخل_ والدین می‌روند.
- **ترتیب بستن**: ابتدا درونی‌ترین را ببندید (LIFO):

  ```html
  <div>
    <p>متن <strong>پررنگ</strong></p>
  </div>
  ```

- مثال **نامعتبر** (انجام ندهید):

  ```html
  <p>متن <strong>اوه</p></strong>
  ```

- **اعتبارسنجی**: از **لینت نیکود** یا [validator.w3.org](https://validator.w3.org) برای تشخیص زودهنگام اشتباهات استفاده کنید.

> **تمرین کوچک (۱۰ دقیقه):** یک صفحه دو بخشی با عناوین، پاراگراف‌ها، لیست‌ها و یک تصویر بسازید—**بدون CSS**. با **لینت نیکود** اعتبارسنجی کنید و امتیاز خود را در انجمن جامعه ارسال کنید.

## 👓 مرحله ۲ · نشانه‌های سند

HTML5 عناصر **نشانه** را معرفی کرد تا فناوری کمکی بتواند مستقیماً به بخش‌ها بپرد. نشانه‌ها به ناوبری کمک می‌کنند.

### ۲.۱ اسکلت معنادار

```html
<header>
  <h1>نمونه کار من</h1>
  <nav aria-label="اصلی">
    <ul>
      <li><a href="#پروژه‌ها">پروژه‌ها</a></li>
      <li><a href="#درباره">درباره</a></li>
      <li><a href="#تماس">تماس</a></li>
    </ul>
  </nav>
</header>

<main id="محتوا">
  <section id="پروژه‌ها">
    <h2>پروژه‌های ویژه</h2>
    <article>
      <h3>پروژه اول</h3>
      <p>توضیح کوتاه...</p>
    </article>
  </section>

  <section id="درباره">
    <h2>درباره من</h2>
    <p>چیزی درباره خودتان بنویسید.</p>
  </section>
</main>

<aside>
  <h2>لینک‌های مرتبط</h2>
  <ul>
    <li><a href="https://example.com">منبع خارجی</a></li>
    <li><a href="blog.html">وبلاگ</a></li>
  </ul>
</aside>

<footer>
  <p>© ۲۰۲۵ نام شما</p>
</footer>
```

**قوانین سرانگشتی**

۱. فقط _یک_ `<main>` در هر صفحه.
۲. هر نشانه (`<header>`، `<nav>`، `<main>`، `<aside>`، `<footer>`، `<form>` اگر مستقل باشد) باید از طریق _Rotor_ در VoiceOver قابل کشف باشد.
۳. از `<section>` / `<article>` برای گروه‌هایی که عنوان خودشان را دارند استفاده کنید.
۴. از `<aside>` برای محتوای حاشیه‌ای استفاده کنید (مثل نوار کناری، لینک‌های مرتبط).

> **غور عمیق نیکود:** آزمون تعاملی نشانه با پیش‌نمایش فوری صفحه‌خوان.

---

### ۲.۲ `<nav>` و لینک‌های پرش

- ناوبری اصلی را در `<nav>` بپیچید.
- اگر چندین nav وجود دارد `aria-label` اضافه کنید:

  ```html
  <nav aria-label="ناوبری فوتر">
    <ul>
      …
    </ul>
  </nav>
  ```

- **لینک پرش**: به کاربران کیبورد اجازه دهید سریع به محتوای اصلی بپرند.

  ```html
  <a href="#محتوا" class="skip-link">پرش به محتوا</a>
  ```

- **CSS** (فقط روی فوکوس نمایش دهید):

  ```css
  .skip-link {
    position: absolute;
    top: -40px;
    left: 0;
    background: #fff;
    padding: 8px;
    z-index: 100;
  }
  .skip-link:focus {
    top: 0;
  }
  ```

> **نکته**: لینک پرش را به عنوان اولین فرزند `<body>` قرار دهید.

## 🧭 مرحله ۳ · ساختار متن و عناوین

### ۳.۱ سلسله مراتب عناوین

```html
<h1>وبلاگ دستور پخت</h1>

<section>
  <h2>کیک شکلاتی</h2>
  <h3>مواد</h3>
  <ul>
    <li>۱ فنجان آرد</li>
    <li>۲ تخم مرغ</li>
    <li>۱/۲ فنجان شکر</li>
  </ul>
  <h3>مراحل</h3>
  <ol>
    <li>فر را گرم کنید...</li>
    <li>مواد را مخلوط کنید...</li>
    <li>۳۰ دقیقه بپزید...</li>
  </ol>
</section>

<section>
  <h2>تارت لیمو</h2>
  <h3>مواد</h3>
  <h3>مراحل</h3>
</section>
```

- یک—و فقط یک—`<h1>` در هر صفحه. ✅
- ترتیب را دنبال کنید: سطوح را نپرید (`<h1>` → `<h3>`). ❌
- عناوین باید منطقی برای طرح کلی تودرتو شوند: `<h1>`، سپس `<h2>`، سپس `<h3>`، و غیره.
- از `role="heading"` استفاده نکنید مگر اینکه ویجت بسیار پیچیده‌ای داشته باشید. عناوین بومی برنده می‌شوند.

> **امتحان کنید:** کد خود را در **طرح کلی نیکود** (آنلاین) بچسبانید تا درخت را تجسم کنید.

### ۳.۲ پاراگراف‌ها، لیست‌ها و متن درون‌خطی

**پاراگراف (`<p>`)**

```html
<p>
  HTML ستون فقرات هر صفحه وب است. جملات ساده را در
  <strong>پاراگراف‌ها</strong> بنویسید.
</p>
```

**لیست نامرتب (`<ul>`)**

```html
<ul>
  <li>مورد اول</li>
  <li>مورد دوم</li>
  <li>مورد سوم</li>
</ul>
```

**لیست مرتب (`<ol>`)**

```html
<ol>
  <li>مرحله اول</li>
  <li>مرحله دوم</li>
  <li>مرحله سوم</li>
</ol>
```

**لیست توضیحات (`<dl>`)**

```html
<dl>
  <dt>HTML</dt>
  <dd>زبان نشانه‌گذاری ابرمتن</dd>
  <dt>CSS</dt>
  <dd>صفحات استایل آبشاری</dd>
</dl>
```

**تگ‌های متن درون‌خطی**

- `<strong>`: متن مهم (اغلب پررنگ).
- `<em>`: تأکید (اغلب ایتالیک).
- `<small>`: متن ریزتر.
- `<mark>`: متن برجسته شده.
- `<time datetime="۲۰۲۵-۰۶-۰۴">۴ خرداد ۱۴۰۴</time>`: تاریخ/زمان با فرمت قابل خواندن ماشین.

> **بهترین شیوه**: از لیست‌ها زمانی استفاده کنید که ترتیب یا گروه‌بندی مهم است. صفحه‌خوان‌ها تعداد موارد را اعلام می‌کنند.

## ✍️ مرحله ۴ · فرم‌های دسترسی‌پذیر و ورودی کاربر

فرم‌ها برای تعامل حیاتی هستند. هر کنترل باید برچسب قابل مشاهده یا پنهان داشته باشد.

### ۴.۱ ساختار فرم پایه

```html
<form action="/اشتراک" method="post" novalidate>
  <fieldset>
    <legend>خبرنامه ایمیل</legend>

    <label for="email">آدرس ایمیل</label>
    <input
      type="email"
      id="email"
      name="email"
      required
      aria-describedby="email-hint"
      placeholder="شما@مثال.com"
    />
    <p id="email-hint">ما هرگز ایمیل شما را به اشتراک نمی‌گذاریم.</p>

    <button type="submit">اشتراک</button>
  </fieldset>
</form>
```

- **`<fieldset>` + `<legend>`**: کنترل‌های مرتبط را گروه‌بندی می‌کند.
- **`<label for>` ↔ `id`**: برچسب و فیلد را متصل می‌کند.
- **`aria-describedby`**: راهنمای اضافی یا پیام خطا را لینک می‌کند.
- **ویژگی `required`**: به مرورگر و AT (فناوری کمکی) می‌گوید اجباری است.
- از `type="email"`، `type="number"` و غیره برای اعتبارسنجی ورودی استفاده کنید.

---

### ۴.۲ ورودی متن، textarea و select

**ورودی متن**

```html
<label for="username">نام کاربری</label>
<input type="text" id="username" name="username" required />
```

**Textarea**

```html
<label for="bio">بیو کوتاه</label>
<textarea
  id="bio"
  name="bio"
  rows="4"
  cols="50"
  placeholder="درباره خودتان به ما بگویید."
></textarea>
```

**منوی انتخاب**

```html
<label for="country">کشور</label>
<select id="country" name="country">
  <option value="">یکی انتخاب کنید</option>
  <option value="ir">ایران</option>
  <option value="us">ایالات متحده</option>
</select>
```

- همیشه `<label>` برای `<input>` و `<textarea>` شامل کنید.
- برای فرم‌های پیچیده، با `<fieldset>` گروه‌بندی کنید.
- از `<option>` داخل `<select>` استفاده کنید. اولین `<option>` می‌تواند یک placeholder باشد.

---

### ۴.۳ چک‌باکس‌ها و دکمه‌های رادیویی

**گروه چک‌باکس**

```html
<fieldset>
  <legend>اشتراک در خبرنامه‌ها</legend>
  <input type="checkbox" id="news" name="newsletter" value="news" />
  <label for="news">اخبار</label>
  <input type="checkbox" id="updates" name="newsletter" value="updates" />
  <label for="updates">به‌روزرسانی محصول</label>
</fieldset>
```

**گروه دکمه رادیویی**

```html
<fieldset>
  <legend>روش تماس ترجیحی</legend>
  <input type="radio" id="contact-email" name="contact" value="email" checked />
  <label for="contact-email">ایمیل</label>
  <input type="radio" id="contact-phone" name="contact" value="phone" />
  <label for="contact-phone">تلفن</label>
</fieldset>
```

- **گروه‌بندی** چک‌باکس‌ها یا رادیوها داخل یک `<fieldset>` با یک `<legend>`.
- هر `<input>` به `<label>` خودش نیاز دارد.
- صفحه‌خوان‌ها legend را قبل از لیست کردن گزینه‌ها اعلام می‌کنند.

---

### ۴.۴ اعتبارسنجی فرم و پیام‌های خطا

۱. از اعتبارسنجی HTML5 (`required`، `type="email"`، `minlength`، `maxlength`) استفاده کنید.
۲. **پیام‌های خطای درون‌خطی** با استفاده از `aria-live` یا `role="alert"` ارائه دهید.

```html
<div id="username-error" role="alert" aria-live="assertive"></div>
<label for="username">نام کاربری</label>
<input
  type="text"
  id="username"
  name="username"
  aria-describedby="username-error"
/>
```

۳. پس از ارسال فرم، خطاها را کنار فیلدها نمایش دهید، مثلاً:

```html
<p id="username-error">نام کاربری الزامی است.</p>
```

> **چالش نیکود:** یک فرم جعلی پر از الگوهای ضد را به یک گوهر WCAG-compliant تبدیل کنید. نمره‌دهی خودکار شامل است.

## 🖼️ مرحله ۵ · تصاویر، شکل‌ها و متن جایگزین

تصاویر به `alt` معنادار نیاز دارند یا اگر تزئینی هستند باید پنهان شوند.

### ۵.۱ الگوهای ویژگی `alt`

| هدف                 | `alt` صحیح                                                       |
| ------------------- | ---------------------------------------------------------------- |
| **اطلاعاتی**        | `alt="چهار عضو تیم در مقابل یک تخته سفید ایستاده‌اند"`           |
| **تزئینی**          | `alt="" role="presentation"`                                     |
| **عملکردی**         | `alt="دانلود بروشور PDF"`                                        |
| **پیچیده (نمودار)** | از `<figure>` + `<figcaption>` و لینک توضیح طولانی استفاده کنید: |

```html
<figure>
  <img src="نمودار.png" alt="نمودار فروش" />
  <figcaption>
    فروش در Q1 ۲۰٪ افزایش یافت. <a href="#longdesc">توضیح طولانی</a>
  </figcaption>
</figure>
<div id="longdesc">
  جزئیات کامل: نمودار نشان می‌دهد فروش Q1 به دلیل … افزایش یافت
</div>
```

**قوانین**

۱. اگر تصویر _معنی_ اضافه می‌کند، یک توضیح واضح و مختصر بنویسید.
۲. اگر نه، از فناوری کمکی پنهان کنید: `alt="" role="presentation"`.
۳. از "تصویر..." یا "عکس..." در alt اجتناب کنید—صفحه‌خوان‌ها "تصویر" را خودکار اعلام می‌کنند.

---

### ۵.۲ تصاویر واکنش‌گرا و High-DPI

```html
<img
  src="عکس-کوچک.jpg"
  srcset="عکس-کوچک.jpg 480w, عکس-متوسط.jpg 800w, عکس-بزرگ.jpg 1200w"
  sizes="(max-width: 600px) 480px, (max-width: 1200px) 800px, 1200px"
  alt="منظره کوهستان"
/>
```

- **`srcset`**: لیست فایل‌های تصویر با توصیف‌کننده‌های عرض (`w`).
- **`sizes`**: به مرورگر می‌گوید کدام تصویر را در عرض‌های مختلف viewport انتخاب کند.
- بارگذاری سریع و تصاویر تیز روی صفحه‌های high-DPI را تضمین می‌کند.

## 🎵 مرحله ۶ · چندرسانه‌ای، لینک‌ها و جداول

### ۶.۱ صدا و ویدیو با زیرنویس

```html
<video controls>
  <source src="video.mp4" type="video/mp4" />
  <track
    kind="captions"
    src="captions_fa.vtt"
    srclang="fa"
    label="فارسی"
    default
  />
  مرورگر شما از ویدیو HTML5 پشتیبانی نمی‌کند.
</video>

<audio controls>
  <source src="audio.mp3" type="audio/mpeg" />
  مرورگر شما از عنصر صدا پشتیبانی نمی‌کند.
</audio>
```

- همیشه `controls` را شامل کنید.
- برای ویدیو، حداقل یک مسیر زیرنویس (`<track kind="captions">`) ارائه دهید.
- برای فقط صدا، ارائه رونوشت در متن را در نظر بگیرید.

---

### ۶.۲ لینک‌ها و ناوبری دسترسی‌پذیر

```html
<a href="about.html">درباره ما</a>
<a href="https://example.com" target="_blank" rel="noopener noreferrer">
  لینک خارجی (در تب جدید باز می‌شود)
</a>
```

**بهترین شیوه‌ها**

- **متن لینک توصیفی**: "اینجا کلیک کنید" مبهم است. از "سیاست حریم خصوصی ما را بخوانید" استفاده کنید.
- **`target="_blank"`**: اگر در تب جدید باز می‌شود، `rel="noopener noreferrer"` اضافه کنید.
- از `title="..."` به ندرت استفاده کنید. فقط زمانی که متن لینک به تنهایی مبهم است.

---

### ۶.۳ جداول برای داده‌های جدولی

```html
<table>
  <caption>
    داده‌های فروش ماهانه
  </caption>
  <thead>
    <tr>
      <th scope="col">ماه</th>
      <th scope="col">درآمد</th>
      <th scope="col">هزینه‌ها</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">فروردین</th>
      <td>۱۰,۰۰۰ تومان</td>
      <td>۷,۰۰۰ تومان</td>
    </tr>
    <tr>
      <th scope="row">اردیبهشت</th>
      <td>۱۲,۰۰۰ تومان</td>
      <td>۸,۰۰۰ تومان</td>
    </tr>
  </tbody>
</table>
```

- `<caption>`: توضیح مختصر هدف جدول.
- از `<thead>`، `<tbody>`، `<tfoot>` برای ساختار استفاده کنید.
- `scope="col"` یا `scope="row"` برای سلول‌های هدر تا صفحه‌خوان‌ها روابط را بدانند.
- از جداول برای layout استفاده نکنید. از CSS استفاده کنید.

## 🚦 مرحله ۷ · برچسب‌گذاری ARIA و مدیریت فوکوس

ARIA دسترسی‌پذیری اضافی اضافه می‌کند وقتی HTML به تنهایی نمی‌تواند آن را مدیریت کند. به ندرت استفاده کنید.

### ۷.۱ ویژگی‌های ARIA رایج

| نیاز                      | ویژگی                | مثال کد                                                           |
| ------------------------- | -------------------- | ----------------------------------------------------------------- |
| برچسب پنهان (دکمه آیکون)  | `aria-label`         | `<button aria-label="بستن">✖</button>`                            |
| برچسب قابل مشاهده خارجی   | `aria-labelledby`    | `<table aria-labelledby="قیمت‌گذاری">…</table>`                   |
| راهنمای اضافی یا پیام خطا | `aria-describedby`   | `<input aria-describedby="email-hint" />`                         |
| نقش پویا (منطقه زنده)     | `aria-live="polite"` | `<div aria-live="polite">در حال بارگذاری به‌روزرسانی‌ها...</div>` |

> **ترتیب خواندن**:
>
> ۱. `aria-labelledby`
> ۲. `<label>` قابل مشاهده
> ۳. `aria-describedby`

---

### ۷.۲ استایل‌های فوکوس و دسترسی‌پذیری کیبورد

```css
:where(a, button, input, select, textarea, [tabindex]) :focus-visible {
  outline: 3px solid #004bcc;
  outline-offset: 2px;
}
```

- **`:focus-visible`** اطمینان می‌دهد استایل فوکوس فقط هنگام ناوبری از طریق کیبورد نمایش داده شود.
- همیشه اطمینان حاصل کنید عناصر تعاملی (لینک‌ها، دکمه‌ها، کنترل‌های فرم) با **Tab** قابل دسترسی هستند.
- outline را بدون جایگزینی با یک استایل قابل مشاهده حذف نکنید—کاربران کیبورد به آن نیاز دارند.

## 🔧 مرحله ۸ · بخش‌های اضافی HTML و بهترین شیوه‌ها

فراتر از عناصر اصلی، اینجا تگ‌های بیشتری هستند که هر روز استفاده خواهید کرد. ما مثال‌های ساده و دلایل ارائه می‌دهیم.

### ۸.۱ متادیتا و عناصر `<head>`

```html
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />
  <meta name="description" content="راهنمای HTML معنادار و دسترسی‌پذیری" />
  <link rel="stylesheet" href="styles.css" />
  <link rel="icon" href="favicon.ico" type="image/x-icon" />
  <title>راهنمای HTML معنادار</title>
</head>
```

- `<meta name="description">`: خلاصه برای موتورهای جستجو.
- `<link rel="stylesheet">`: فایل‌های CSS را لینک کنید.
- `<link rel="icon">`: آیکون سایت در تب‌های مرورگر.
- `<meta name="author">`، `<meta name="keywords">` را در نظر بگیرید (اگرچه امروز برای سئو کمتر حیاتی هستند).

---

### ۸.۲ جاسازی SVG و آیکون‌ها

مثال SVG درون‌خطی:

```html
<svg
  width="24"
  height="24"
  viewBox="0 0 24 24"
  aria-hidden="true"
  focusable="false"
>
  <path d="M12 2L2 22h20L12 2z" />
</svg>
```

- برای SVG‌های تزئینی، `aria-hidden="true"` اضافه کنید.
- اگر SVG معنی منتقل می‌کند، `<title>` داخل `<svg>` شامل کنید.

مثال با عنوان:

```html
<svg
  width="100"
  height="100"
  viewBox="0 0 100 100"
  role="img"
  aria-labelledby="triangleTitle"
>
  <title id="triangleTitle">آیکون مثلث هشدار</title>
  <path d="M50 5 L95 95 H5 Z" fill="orange" />
</svg>
```

---

### ۸.۳ جاسازی محتوای خارجی

**ویدیو یوتیوب** (با بهبود حریم خصوصی):

```html
<iframe
  width="560"
  height="315"
  src="https://www.youtube-nocookie.com/embed/VIDEO_ID"
  title="پخش‌کننده ویدیو یوتیوب"
  frameborder="0"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
  allowfullscreen
>
</iframe>
```

- از `title="..."` روی `<iframe>` استفاده کنید تا صفحه‌خوان‌ها هدف را بدانند.
- دامنه "nocookie" را برای کاهش ردیابی ترجیح دهید.

---

### ۸.۴ بهبود تدریجی و پشتیبانی بدون JS

وقتی سرانجام جاوااسکریپت اضافه می‌کنید، همیشه یک پشتیبانی HTML/CSS کارآمد ارائه دهید:

```html
<button id="menu-toggle">منو</button>
<nav id="site-nav">
  <ul>
    <li><a href="/">خانه</a></li>
    <!-- لینک‌های بیشتر -->
  </ul>
</nav>

<script>
  // جاوااسکریپت کلاس "open" را تغییر می‌دهد
  document.getElementById("menu-toggle").addEventListener("click", () => {
    document.getElementById("site-nav").classList.toggle("open");
  });
</script>
```

- بدون JS، منو همچنان قابل مشاهده است.
- با JS، آن را با نمایش/پنهان و انیمیشن‌ها بهبود می‌دهید.

## 🛠️ پروژه نهایی · "درباره من" دسترسی‌پذیر خود را بسازید (فقط HTML)

۱. **نشانه‌های معنادار**: `<header>`، `<nav>`، `<main>`، `<aside>` (اختیاری)، `<footer>`.
۲. **عکس پروفایل**: از `<img src="..." alt="متن توصیفی" />` استفاده کنید.
۳. **فرم تماس**:

- فیلدهای ایمیل + پیام.
- از `<label>`، `<fieldset>` و `aria-describedby` درست استفاده کنید.

۴. **پالت رنگ WCAG AA**: از **بررسی‌کننده کنتراست نیکود** برای انتخاب رنگ‌ها استفاده کنید.
۵. **سطوح عنوان درست**: `<h1>` → `<h3>`، هرگز نپرید.
۶. **استایل‌های فوکوس سفارشی**: از طریق CSS.
۷. **تصویر واکنش‌گرا**: از `srcset` + `sizes` استفاده کنید.
۸. **لینک‌های خارجی**: همه در همان تب باز می‌شوند مگر اینکه ضروری باشد؛ برای تب‌های جدید `rel="noopener noreferrer"` اضافه کنید.
۹. **آیکون SVG**: یک SVG درون‌خطی با `aria-hidden` یا `<title>` درست شامل کنید.

> **انتشار** در GitHub Pages. URL را در **نمایشگاه نیکود** برای بازخورد ارسال کنید.

## 🎒 تکالیف و تمرین

برای تثبیت دانش خود، این تمرینات را کامل کنید و مدرک را در **نیکود** ارسال کنید:

۱. **مینی‌سایت: "درباره من"**

- یک سایت تک صفحه‌ای با تمام نیازمندی‌های پروژه نهایی بسازید.
- مدرک: URL زنده + ویدیو کوتاه راه‌روم صفحه‌خوان (نشان دادن پرش بین نشانه‌ها).

۲. **مقاله خبری معنادار**

- یک صفحه مقاله (بدون `<div>` برای ساختار) فقط با تگ‌های معنادار ایجاد کنید.
- حداقل یک مقاله، دو بخش، یک شکل با عنوان، و یک جدول (داده‌های جدولی) شامل کنید.
- مدرک: لینک مخزن + اسکرین‌شات از طرح کلی HTML.

۳. **فرم تماس دسترسی‌پذیر**

- فرمی برای رزرو تور طراحی کنید: نام، ایمیل، تاریخ (از `<input type="date">` استفاده کنید)، تعداد مهمانان (از `<select>` استفاده کنید)، و پیام (`<textarea>`).
- با ویژگی‌های HTML5 (`required`، `type="email"`) اعتبارسنجی کنید، و پیام‌های خطا را درون‌خطی با `aria-live` نمایش دهید.
- مدرک: اسکرین‌شات از فرم با خطای نمایش داده شده + قطعه کد.

۴. **تمرین تصاویر و متن جایگزین**

- سه تصویر انتخاب کنید:

  ۱. نمودار اطلاعاتی → `alt` بنویسید که اطلاعات کلیدی را توصیف کند.
  ۲. گرافیک تزئینی → از AT پنهان کنید (`alt="" role="presentation"`).
  ۳. آیکون عملکردی (مثل چاپ یا دانلود) → `alt` توصیفی.

- مدرک: قطعه کد از هر تگ `<img>` با توضیح.

۵. **کنتراست رنگ WCAG AA**

- پالت حداقل سه ترکیب متن/پس‌زمینه ایجاد کنید.
- از **بررسی‌کننده کنتراست نیکود** یا دیگری استفاده کنید.
- مدرک: اسکرین‌شات از نتایج بررسی‌کننده + کدهای هگز رنگ‌ها.

۶. **آزمون HTML** (امتیاز اضافی)

- آزمون کوچک درست/غلط در معناداری HTML را کامل کنید:

  - "`<div>` برای عناوین بخش توصیه می‌شود." (غلط)
  - "پریدن از سطوح عنوان اگر از نظر بصری تمیز باشد قابل قبول است." (غلط)
  - "`<main>` باید در هر صفحه منحصر به فرد باشد." (درست)

- مدرک: پاسخ‌ها + توضیح کوتاه (۱–۲ جمله هر کدام).

> **امتیاز اضافی**: یک مثال واکنش‌گرا `<picture>` با پشتیبانی WebP ایجاد کنید:
>
> ```html
> <picture>
>   <source type="image/webp" srcset="image.webp" />
>   <source type="image/jpeg" srcset="image.jpg" />
>   <img src="image.jpg" alt="توضیح جایگزین" />
> </picture>
> ```
>
> توضیح دهید چرا مرورگرها فرمت‌های مختلف انتخاب می‌کنند.
