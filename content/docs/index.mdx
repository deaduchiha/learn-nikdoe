# 🌳 Zero‑to‑Hero Frontend

Hey there! This guide is built from years of helping total beginners become frontend pros. It's all about really understanding the concepts and building cool stuff along the way. Go at your own pace—move forward when things click, not just because a schedule says so.

> 💡 **How to progress:** Pick one concept at a time, build something small with it, then iterate. Your pace is yours alone.

---

## 🌱 0 · Roots — Tools & Foundations

| Skill Targets         | Hands‑On Practice                                                                                                                            | Curated Resources                                                                                                                                                                                                                          |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Dev environment ready | • Install **VS Code**, **Node LTS**, **Git**.<br />• Enable Chrome / Firefox DevTools.<br />• Create a GitHub repo & push your first commit. | • [VS Code “Get Started”](https://code.visualstudio.com/docs/introvideos/basics)<br />• [Oh My Zsh](https://ohmyz.sh/) (CLI comfort)<br />• [GitHub Hello World Quickstart](https://docs.github.com/en/get-started/quickstart/hello-world) |

**Checkpoint 🔎** — You can clone, edit, commit, and push without reading a cheat‑sheet.

---

## 🌳 1 · TRUNK — HTML & Accessibility

- **Concepts:** semantic landmarks, forms, headings hierarchy, `alt`, ARIA roles, focus management, logical reading order.
- **Mini‑Project:** One‑page **“About Me”** with `<header>`, `<nav>`, `<main>`, `<footer>`; pass **WCAG AA** colour contrast.
- **Guides:** [MDN HTML Elements](https://developer.mozilla.org/en-US/docs/Web/HTML/Element), [Web.dev Accessibility](https://web.dev/learn/accessibility/).

---

## 🌿 2 · BIG BRANCH — CSS Essentials

- **Concepts:** cascade & inheritance, box‑model, selectors & specificity, Flexbox, Grid, **CSS Nesting & Layers** (2025), logical properties, media & container queries, BEM / utility naming.
- **Mini‑Project:** Recreate a product card from an ecommerce site pixel‑perfect; make it fluid down to 320 px.
- **Guides:** [Kevin Powell Flexbox + Grid](https://www.youtube.com/@KevinPowell), [Modern CSS—Stephanie Eckles](https://moderncss.dev/).

---

## 🍃 3 · BRANCHES — JavaScript Fundamentals

- **Concepts:** ES2024 syntax (modules, optional chaining, nullish coalescing), data types, scope & closures, arrays / objects methods, destructuring & spread, template literals, DOM, events, fetch API, async/await, JSON.
- **Mini‑Projects:**

  1. **Rock–Paper–Scissors** in vanilla JS + DOM.
  2. **Weather Dashboard** — fetch OpenWeather API, show 5‑day forecast with loading & error states.

- **Guides:** [JavaScript.info](https://javascript.info/), [*Eloquent JavaScript* (online)](https://eloquentjavascript.net).

> ✅ **Discipline:** Commit early & often, write self‑documenting code, enable **ESLint + Prettier** on save.

---

## 🌸 4 · BLOSSOM — Modern Layout & Design Systems

- **Topics:** Custom properties, `calc()` / `clamp()`, fluid type & spacing scales, dark‑mode tokens, responsive design with **Tailwind CSS v4** and **CSS functions**.
- **Mini‑Project:** Refactor Weather Dashboard to Tailwind; implement system dark mode (`prefers‑color‑scheme`) and build a small component library using **shadcn/ui**.
- **Guides:** [Tailwind Docs](https://tailwindcss.com/docs), [Smashing Mag Fluid Type](https://www.smashingmagazine.com/2022/01/modern-fluid-typography-css-clamp/).

---

## 🌺 5 · FLOWERS — Framework Fundamentals (React 19)

- **Concepts:** Vite, JSX, props vs state, hooks (`useState`, `useEffect`, `useMemo`), component composition, keys & lists, conditional render, React DevTools.
- **Mini‑Project:** **Todo List SPA** with filter tabs, drag‑and‑drop, and localStorage persistence.
- **Guides:** [react.dev Tutorial](https://react.dev/learn), [Fireship React 19 “in 100 seconds”](https://youtu.be/).

---

## 🌞 6 · CANOPY — Data & State Mastery

| Focus                  | Hands‑On                                                            | References                              |
| ---------------------- | ------------------------------------------------------------------- | --------------------------------------- |
| **Fetching & Caching** | Integrate **TanStack Query v6** with optimistic updates & suspense. | TanStack Query Docs                     |
| **Global State**       | Compare Context, **Zustand**, Jotai; migrate Todo SPA to Zustand.   | Zustand Docs, State‑Management Patterns |
| **Error Boundaries**   | Implement top‑level and route‑level error UI.                       | react.dev Error Boundaries              |
| **Forms**              | Use React Hook Form v9 with zod validation.                         | RHF Docs, Zod Docs                      |

---

## 🌲 7 · BRISTLES — TypeScript & Testing

- **TypeScript:** basic types, interfaces vs types, generics, utility types, React props typing, `ts‑config` strictness.
- **Testing:** **Vitest 2**, **Testing Library**, **Cypress 15** for E2E; practice TDD on Rock–Paper–Scissors.
- **Mini‑Project:** Refactor Todo SPA to TypeScript; achieve 80%+ unit coverage & one happy‑path E2E.
- **Guides:** *Learning TypeScript* (O’Reilly), Testing Library Docs.

---

## 🌟 8 · CONSTELLATION — Career & Portfolio

- **Version control mastery:** interactive rebase, semantic commits, conventional changelog.
- **Design fluency:** Figma inspect, tokens, accessible palettes.
- **Soft skills:** PR etiquette, constructive feedback, Agile rituals.
- **Interview prep:** system design for frontend, STAR stories, mock interviews (CoderPad).

---

## 🎯 Milestone Checklist

- **🌱 Roots:** Dev environment configured, first commit pushed.
- **🌳 Core Web:** “About Me” page validates & meets WCAG AA.
- **🌿 JS Ready:** Weather Dashboard fetches real data with graceful fallbacks.
- **🌺 React Sprout:** Todo SPA works offline and persists state.
